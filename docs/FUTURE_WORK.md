## Filtering

Within v1.0, issues are grouped by phase label:

- `phase-1-foundation` — branding, Apple requirements
- `phase-2-features` — core feature work
- `phase-3-polish` — animations, empty/error states, accessibility
- `phase-4-ship` — legal, listing, TestFlight, review

Example: open Phase 2 features → https://github.com/bgibso4/baseline/issues?q=is%3Aopen+label%3Aphase-2-features

## Why the move

FUTURE_WORK.md was a living checkbox list that repeatedly drifted out of sync with actual work. Individual GitHub issues let us link PRs, attach discussions, track milestones, and filter — things a flat markdown file can't.
